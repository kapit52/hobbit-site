# Ширский уголок — таверна для добрых путников

Полноценный сайт ресторана: PHP 8.2 + MySQL 8.0 + Docker.
Дизайн — Cinzel/Cormorant Garamond, пергаментная палитра, светлячки.

---

## Быстрый запуск через Docker

**Требования:** [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Windows 10/11 с WSL2 или Mac).

```bash
docker compose up --build -d
```

| Адрес | Описание |
|-------|----------|
| http://localhost:8080 | Сайт таверны |
| http://localhost:8080/admin_login.php | Управление (для персонала) |

### Учётные данные

Вход для гостей — **по почте и паролю** (логин/никнейм убраны). Админ может войти и по почте, и по старому логину `admin`.

| Роль | Почта (или логин) | Пароль |
|------|-------------------|--------|
| Администратор | `admin@shire-corner.local` (или `admin`) | `shire123` |
| Демо-гость | `guest@example.com` | `shire123` |

Страница входа админа: `admin_login.php`. Гости входят на `login.php`.

### Остановка

```bash
docker compose down          # остановить, данные сохраняются
docker compose down -v       # полная очистка БД (начать с нуля)
```

## База данных и её хранение

**Данные БД лежат в папке проекта — `docker/mysql-data/`** (bind-mount в
`docker-compose.yml`), а НЕ в именованном Docker-томе. Из этого следует главное:

- ✅ `docker compose down -v` **НЕ стирает базу** — `-v` удаляет только именованные
  тома, а папку с данными не трогает. Картинки и любые правки никуда не денутся.
- ✅ Папка проекта самодостаточна: скопировал её на другой ПК — приехала и БД, и фото.

Первый запуск **на пустой** `docker/mysql-data/` (свежий клон без этой папки) MySQL
сам наполняет из снимка `docker/init/01_database.sql` (полная схема + данные: меню,
фото-привязки слотов, отзывы, аккаунты). Никаких ручных шагов не нужно.

### Перенос проекта на другой ПК

Скопируй всю папку проекта и запусти:
```
docker compose up --build -d
```
Что должно приехать вместе с папкой:
- `public/uploads/` — сами файлы фотографий;
- `docker/mysql-data/` — база со всеми привязками (если папки нет — БД соберётся из снимка).

### Полный сброс базы «с нуля»

```
docker compose down
# удалить папку docker/mysql-data  (PowerShell: Remove-Item -Recurse -Force docker/mysql-data)
docker compose up -d
```
На пустой `docker/mysql-data/` база заново соберётся из снимка `docker/init/01_database.sql`.

### Снимок-сид (опционально)

`docker/init/01_database.sql` — это сид для чистого старта. Он нужен только когда
`docker/mysql-data/` пустая. Чтобы обновить сид под текущее состояние БД (например, перед
тем как поделиться чистым клоном без `mysql-data`), запусти при поднятом стеке:
```
docker/snapshot-db.ps1      # Windows
sh docker/snapshot-db.sh    # Linux/Mac
```
